DDA Algorithm

Digital Differential Analyzer: A step-by-step line drawing technique.

How it Works

DDA uses floating-point arithmetic to calculate the next pixel on a line by incrementally adding the slope to either x or y.

  • Compute dx = x2 - x1 and dy = y2 - y1
  • steps = max(|dx|, |dy|)
  • xInc = dx / steps, yInc = dy / steps

Characteristics

Advantages: Simple to understand and implement.

Disadvantages: Uses slow floating-point arithmetic. Rounding errors accumulate.

Complexity: O(steps) per line.

Complexity & Exam Tips

Time Complexity: O(N)
Where N is the number of steps (max of dx or dy).
Space Complexity: O(1)
Only a few variables are stored.
Advantages: Simple. Eliminates loop multiplication.
Disadvantages: Floating point arithmetic is slow. Rounding error.
Exam Tip: Examiners love asking why DDA is slower than Bresenham. Mention floating-point addition and the `round()` function overhead.

Bresenham's Line Algorithm

Integer-based, highly efficient line drawing algorithm.

Decision Parameter

Bresenham's algorithm uses a decision parameter P to choose between the East (E) or North-East (NE) pixel.

  • Initial P₀: 2dy - dx
  • If P < 0: Choose E pixel. P_{k+1} = P_k + 2dy
  • If P ≥ 0: Choose NE pixel. P_{k+1} = P_k + 2dy - 2dx

Complexity & Exam Tips

Time Complexity: O(N)
N is dx (the driving axis).
Space Complexity: O(1)
Uses only integer variables.
Advantages: Fast, integer-only. No rounding errors. Exact pixels.
Disadvantages: More complex initial logic than DDA.
Exam Tip: Be ready to trace the Decision Parameter (Pk) table. If Pk is negative, only X increments. If Pk is positive, both X and Y increment!

Line Clipping (Cohen-Sutherland)

Efficiently removing lines or parts of lines outside the viewing window.

Outcodes

Every point is assigned a 4-bit binary code (TBRL) indicating its position relative to the clipping window (Top, Bottom, Right, Left).

1001 | 1000 | 1010
------------------
0001 | 0000 | 0010
------------------
0101 | 0100 | 0110

Algorithm Steps

  1. Compute outcodes for P1 and P2.
  2. Trivially Accept: Both outcodes are 0000.
  3. Trivially Reject: Logical AND of outcodes != 0000.
  4. Clip: Find intersection with window boundary and repeat.

Complexity & Exam Tips

Time Complexity: O(1) to O(K)
Very fast for trivial cases.
Space Complexity: O(1)
Outcodes are just 4 bits.
Advantages: Extremely fast for trivially accepted/rejected lines.
Disadvantages: Needs intersection math if not trivial.
Exam Tip: The Outcode sequence is always TBRL (Top, Bottom, Right, Left). Memorize that `AND != 0000` means trivial rejection.

Polygon Clipping (Sutherland-Hodgman)

Clipping an entire polygon against each edge of a window.

Four Cases for Each Edge

When traversing from vertex V1 to V2 against a clipping boundary:

  • Inside → Inside: Add V2.
  • Inside → Outside: Add Intersection (I).
  • Outside → Outside: Add nothing.
  • Outside → Inside: Add Intersection (I) and V2.

Algorithm Flow

The entire polygon is clipped against the Left edge, producing a new polygon. This new polygon is then clipped against the Right edge, then Top, then Bottom.

Complexity & Exam Tips

Time Complexity: O(N * E)
N vertices against E clip edges.
Space Complexity: O(N)
Intermediate polygons must be stored.
Advantages: Works well for convex clipping regions. Simple pipeline.
Disadvantages: Creates extraneous edges for concave polygons.
Exam Tip: Know the 4 intersection cases perfectly! Outside → Inside generates TWO points (Intersection + V2).

Lighting & Shading

Modeling realistic appearance using the Phong Reflection Model.

Phong Reflection Model

Combines three components to simulate light interacting with a surface:

  • Ambient: Constant background light scattered everywhere.
  • Diffuse: Light scattered equally in all directions (matte).
  • Specular: Highlight from reflected light (shiny).
Total = Ambient + Diffuse + Specular

Shading Models

  • Flat Shading: One normal per polygon. Blocky appearance.
  • Gouraud (Smooth): Normals at vertices, interpolate colors.
  • Phong Shading: Interpolate normals, compute lighting per pixel. Best quality.

Complexity & Exam Tips

Time Complexity: O(P)

Texture Mapping

Wrapping a 2D image around a 3D object to add realistic detail.

UV Coordinates

Textures use a 2D coordinate system (U, V) ranging from 0.0 to 1.0. These are mapped to the (X, Y, Z) vertices of 3D geometry.

(U=0, V=0) = Bottom-Left
(U=1, V=1) = Top-Right

OpenGL Workflow

  • glGenTextures() - Generate texture ID
  • glBindTexture() - Make texture active
  • glTexImage2D() - Load image data
  • glEnable(GL_TEXTURE_2D) - Turn on mapping

Complexity & Exam Tips

Time Complexity: O(P)
Per-pixel texture lookup.
Space Complexity: O(T)
Where T is the size of the texture image in memory.
Advantages: Adds immense realism without geometry overhead.
Disadvantages: Can look pixelated up close (needs filtering).
Exam Tip: Always remember UV coordinates go from 0.0 to 1.0 regardless of actual image pixel size.

Why do they look different?

DDA uses round() which can pull the pixel selection in slightly different directions compared to Bresenham's exact integer Pk decision, especially on lines with specific fractional slopes. Bresenham is considered the mathematically "truer" representation of the ideal line.

Common Viva Questions

Be prepared for direct examiner questions.

Q: Why is DDA less accurate than Bresenham?

A: DDA uses floating-point arithmetic. Successive additions of fractional numbers (xInc, yInc) cause rounding errors to accumulate, eventually pushing the plotted pixel off the ideal mathematical line. Bresenham uses exact integer mathematics.

Q: Why is Bresenham faster?

A: It relies entirely on integer addition, subtraction, and bit-shifting (multiplying by 2). It avoids expensive floating-point operations and the `round()` function.

Q: What is an outcode in Cohen-Sutherland?

A: A 4-bit binary code assigned to a point that describes its position relative to the viewing window (Top, Bottom, Right, Left). It allows for rapid bitwise AND/OR operations to accept or reject lines.

Q: Explain the difference between Flat, Gouraud, and Phong shading.

A:
- Flat: Calculates light once per polygon. Very blocky.
- Gouraud: Calculates light at vertices and interpolates colors across the polygon. Smoother, but misses specular highlights in the center of large polygons.
- Phong: Interpolates normal vectors across the polygon and calculates the full lighting equation for every single pixel. Most realistic, most expensive.

Q: What happens if the Aspect Ratio of the World Window doesn't match the Viewport?

A: The resulting image on the screen will be stretched or squashed (distorted). To maintain proportional geometry, `(Window Width / Window Height)` must equal `(Viewport Width / Viewport Height)`.
